Introduction

Getting started

Rita is a command line application to solve principally modelling problems involving partial differential equations by the OFELI library. It can also be used to solve various basic problems in numerical analysis (Solution of ordinary differential equations, solution of algebraic equation, data fitting, numerical integration, ...)
Rita can be either run interactively or using a script file. This latter option can be chosen simply either by executing
      rita script-file
or once in the Rita execution by typing
      load script-file
The file script-file must contain all commands to be executed, one on each line. It can contain comment lines (lines starting with a #)

Modelling

Mathematical modelling is the procedure that gives to a defined process (physical, mechanical, biological, ...) a mathematical formulation. More precisely, we are concerned here with the case where the mathematical problem consists in a (system of) partial differential equation(s).
Rita is a tool to help solving such a system in a hopefully user-friendly way.
Rita is not itself a numerical software to solve partial differential equations. This one is an interactive interface that uses the OFELI library for solving the defined problem.

Module 'mesh'

A crucial step in the solution procedure of a given problem is the creation of a mesh. This one is the discretization of the physical domain where the process to study takes place. This means partitioning the domain (1-D, 2-D or 3-D) into specific subdomains. More commonly, intervals in 1-D, triangles or quadrilaterals in 2-D, and tetrahedra, hexahedra or pentahedra in 3-D.
RITA does not contain a powerful mesh generator. We use a simple 2-D generator issued from the OFELI library. For 3-D meshes we recommend the use of open source mesh generators like Gmsh or Tetgen.

Once the module mesh is activated, a series of keywords are accessible:

dim <n> Set space dimension: n must be 1, 2 or 3. By default, this value is set to 2. Note that the choice of some keywords below leads automatically to a prescribed dimension
1d Generate a 1-D mesh
rectangle Generate triangulation of a rectangular domain
cube Generate mesh of a cube
vertex Define a vertex in the polygonal domain
line Define a (straight) line in the polygonal domain
circle Define a circle (a portion or the entire one) in the polygonal domain
subdomain Define a subdomain
generate Generate mesh
plot Plot mesh. In current version, we use Gmsh to display mesh
clear Clear generated data for mesh. This enables reinitializing the mesh generation process
read Read mesh or domain data from given file
save Save mesh in file under different formats
end Go back to module menu

Keyword: 1d

This command activates a submodule where the following keywords are available:

interval <xmin> <xmax> Set the interval that defines the domain (Default interval: (0,1))
ne <n> Define the number of elements to generate (Default value: 10)
codes <cmin> <cmax> Define codes (integers) to assign to extreme vertices: (left then right) A nonzero code means that no value will be prescribed on this node (Default codes 0 0)
nbdof <n> Define number of degrees of freedom to associate to each node (Default value: 1)
save [file] Generate mesh and save it in file. If the file name is not provided, mesh is saved in file rita-1d.m
end Return to mesh menu. Note that if the user tries to go back to main menu, without using this keyword, the application will display a warning that no mesh is saved, but one can always go back to keyword 1d and save generated mesh: Nothing is lost

Keyword: rectangle

This command activates a submodule where the following keywords are available:

x <xmin> <xmax> Set min and max values of x: the rectangle is given by (xmin,xmax)*(ymin,ymax)
y <ymin> <ymax> Set min and max values of y: the rectangle is given by (xmin,xmax)*(ymin,ymax)
ne <nx> <ny> Define subdivision on the x- and y-axes
codes <c1> <c2> <c3> <c4> Define codes (integers) to assign to nodes on the lines y=ymin, x=xmax, y=ymax, x=xmin respectively. A nonzero code means that no value will be prescribed on this node (Default codes 0 0 0 0)
nbdof <n> Define number of degrees of freedom to associate to each node (Default value: 1)
save [file] Generate mesh and save it in file. If the file name is not provided, mesh is saved in file rita-rect.m
end Return to mesh menu. Note that if the user tries to go back to main menu, without using this keyword, the application will display a warning that no mesh is saved, but one can always go back to keyword rectangle and save generated mesh: Nothing is lost

Keyword: cube

This command runs a submodule where the following keywords are available:

x <xmin> <xmax> Set min and max values of x: the cube is given by (xmin,xmax)*(ymin,ymax)*(zmin,zmax)
y <ymin> <ymax> Set min and max values of y: the cube is given by (xmin,xmax)*(ymin,ymax)*(zmin,zmax)
z <zmin> <zmax> Set min and max values of z: the cube is given by (xmin,xmax)*(ymin,ymax)*(zmin,zmax)
ne <nx> <ny> <nz> Define subdivision on the x-, y- and z-axes
codes <c1> <c2> <c3> <c4> <c5> <c6> Define codes (integers) to assign to nodes on the lines x=xmin, x=xmax, y=ymin, y=ymax, z=zmin, z=zmax respectively. A nonzero code means that no value will be prescribed on this node (Default codes 0 0 0 0 0 0)
nbdof <n> Define number of degrees of freedom to associate to each node (Default value: 1)
save [file] Generate mesh and save it in file. If the file name is not provided, mesh is saved in file rita-cube.m
end Return to mesh menu. Note that if the user tries to go back to main menu, without using this keyword, the application will display a warning that no mesh is saved, but one can always go back to keyword cube and save generated mesh: Nothing is lost

Keyword: vertex

This command runs a submodule to define vertex data. The following keywords are available:

coord <x> <y> Set x and y coordinates
code <c> Set code for node: Necessary to enforce a boundary condition
size <h> Set size of edges around vertex
save Save vertex
end Return to mesh menu. Note that if the user tries to go back to main menu, without saving vertex, the application will display a warning that no vertex is saved, but one can always go back to keyword vertex and save it: Nothing is lost

Keyword: line

This command runs a submodule to define line data. A line is a straight segments whose ends are defined vertices. The following keywords are available:

vertices <v1> <v2> Define labels of vertices of the line. The line is oriented from v1 to v2
code <c> Set code to assign to generated nodes or sides: If c is positive, its value will be assigned to all generated nodes on line, if it is negative, the value -c will be assigned to all generated sides
save Save line
end Return to mesh menu. Note that if the user tries to go back to main menu, without saving line, the application will display a warning that no line is saved, but one can always go back to keyword line and save it: Nothing is lost

Keyword: circle

This command runs a submodule to define circle data. A line is a straight segments whose ends are defined vertices. The following keywords are available:

vertices <v1> <v2> <v3> Define labels of three vertices. The circular arc goes from vertex v1 to v2. Its center is located at vertex v3. If v1=v2, the arc is the entire circle
code <c> Set code to assign to generated nodes or sides: If c is positive, its value will be assigned to all generated nodes on circular arc, if it is negative, the value -c will be assigned to all generated sides
save Save line
end Return to mesh menu. Note that if the user tries to go back to main menu, without saving circle, the application will display a warning that no circle is saved, but one can always go back to keyword circle and save it: Nothing is lost

Keyword: subdomain

This command runs a submodule to define subdomain data. A subdomain is a region of the domain that can be identified by a line. The following keywords are available:

line <l> Set line number l a lying in subdomain
orientation <s> Set orientation of subdomain. Must be equal to -1 or 1
code <c> Set code to assign to generated elements in subdomain
save Save subdomain
end Return to mesh menu. Note that if the user tries to go back to main menu, without saving defined subdomain, the application will display a warning that no subdomain is saved, but one can always go back to keyword subdomain and save generated subdomain

Keyword: read

This command runs a submodule to read mesh data from a given file. The following keywords are available:

domain <file> Read domain data from Domain file file. Domain file format is defined in the OFELI library
mesh <file> Read mesh data from Mesh file file. Mesh file format is defined in the OFELI library
gmsh <file> Read mesh data from Gmsh file file. Gmsh mesh file format is defined in the Gmsh library
end Return to mesh menu

Keyword: save

This command runs a submodule to save mesh data in a given file under various formats. The following keywords are available:

domain <file> Save domain data in file file. Domain file format is defined in the OFELI library
mesh <file> Save mesh data in Mesh file file. Mesh file format is defined in the OFELI library
gmsh <file> Save mesh data in Gmsh file file. Gmsh mesh file format is defined in the Gmsh library
vtk <file> Save mesh data in vtk file file. vtk file format is defined in the link vtk. It can be useful to display mesh and results using the software Paraview
gnuplot <file> Save mesh data in gnuplot file file. gnuplot file format is useful to display mesh using Gnuplot
matlab <file> Save mesh data in Matlab® file file. Matlab® file format is useful to display mesh using Matlab
tecplot <file> Save mesh data in Tecplot® file file. Tecplot® file format is useful to display mesh using Tecplot
end Return to mesh menu
